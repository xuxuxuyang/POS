# POS机在线申请微信小程序方案设计

## 一、项目概述

### 项目背景
随着移动支付的普及，POS机市场仍有巨大潜力，特别是针对小微商户、个体经营者及特定行业。本项目旨在开发一款微信小程序，简化POS机申请流程，提高转化率，建立长期客户关系。

### 项目目标
1. 打造便捷的POS机在线申请平台
2. 提供良好的用户体验，提高申请转化率
3. 建立完整的客户管理和跟进系统
4. 形成可持续的业务模式和收入来源

### 核心价值
1. 降低获客成本
2. 提高服务效率
3. 建立数据资产
4. 实现业务流程数字化

## 五、数据模型

### 用户信息(user)
```
{
  _id: String,             // 用户ID
  openid: String,          // 微信openid
  phone: String,           // 手机号
  name: String,            // 姓名
  idcard: String,          // 身份证号
  address: Object,         // 地址信息
  business_type: String,   // 经营类型
  created_at: Timestamp,   // 创建时间
  updated_at: Timestamp,   // 更新时间
  status: Number           // 状态：0-未认证，1-已认证
}
```

### 产品信息(product)
```
{
  _id: String,             // 产品ID
  name: String,            // 产品名称
  model: String,           // 型号
  category: String,        // 分类
  images: Array,           // 图片列表
  description: String,     // 描述
  specifications: Object,  // 规格参数
  price: Number,           // 价格
  deposit: Number,         // 押金
  rate: Object,            // 费率信息
  stock: Number,           // 库存
  sales: Number,           // 销量
  status: Number,          // 状态：0-下架，1-上架
  created_at: Timestamp,   // 创建时间
  updated_at: Timestamp    // 更新时间
}
```

### 订单信息(order)
```
{
  _id: String,             // 订单ID
  user_id: String,         // 用户ID
  product_id: String,      // 产品ID
  quantity: Number,        // 数量
  total_amount: Number,    // 总金额
  deposit_amount: Number,  // 押金金额
  deposit_status: Number,  // 押金状态：0-未支付，1-已支付，2-已退还
  delivery_type: Number,   // 配送类型：0-邮寄，1-上门安装
  delivery_address: Object,// 配送地址
  appointment_time: Date,  // 预约时间（上门安装）
  order_status: Number,    // 订单状态：0-待审核，1-审核通过，2-待发货，3-已发货，4-已安装，5-已完成，6-已取消
  logistics_info: Object,  // 物流信息
  merchant_info: Object,   // 商户信息
  bank_info: Object,       // 银行卡信息
  remarks: String,         // 备注
  created_at: Timestamp,   // 创建时间
  updated_at: Timestamp    // 更新时间
}
```

### 支付记录(payment)
```
{
  _id: String,             // 支付记录ID
  order_id: String,        // 订单ID
  user_id: String,         // 用户ID
  amount: Number,          // 金额
  type: Number,            // 类型：0-押金支付，1-押金退还
  transaction_id: String,  // 微信支付交易ID
  status: Number,          // 状态：0-处理中，1-成功，2-失败
  created_at: Timestamp,   // 创建时间
  updated_at: Timestamp    // 更新时间
}
```

### 客户跟进记录(follow_up)
```
{
  _id: String,             // 记录ID
  user_id: String,         // 用户ID
  admin_id: String,        // 管理员ID
  content: String,         // 跟进内容
  contact_type: Number,    // 联系方式：0-电话，1-短信，2-微信
  follow_up_time: Date,    // 跟进时间
  next_follow_up_time: Date, // 下次跟进时间
  created_at: Timestamp,   // 创建时间
  updated_at: Timestamp    // 更新时间
}
```

## 六、用户流程

### 申请流程
1. 用户浏览产品列表
2. 选择合适的POS机产品
3. 点击"立即申请"
4. 填写基本信息（姓名、手机号、地址等）
5. 填写商户信息（经营类型、交易量等）
6. 上传身份证、营业执照等证件
7. 选择配送方式（邮寄）
   - 邮寄：填写收货地址
8. 支付押金
9. 提交申请，等待审核
10. 审核通过后，等待发货/安装
11. 确认收货/安装完成
12. 押金退还

### 管理流程
1. 管理员接收新订单通知
2. 审核订单信息
3. 审核通过后安排发货/安装
4. 更新订单状态
5. 确认安装激活
6. 处理押金退还
7. 跟进客户使用情况
8. 数据分析与报表生成

## 七、API接口设计

### 用户认证类
- `/api/auth/login` - 用户登录
- `/api/auth/phone` - 手机号绑定
- `/api/user/profile` - 获取/更新用户信息
- `/api/user/verify` - 用户身份认证

### 产品管理类
- `/api/products` - 获取产品列表
- `/api/products/:id` - 获取产品详情
- `/api/products/categories` - 获取产品分类
- `/api/products/recommend` - 获取推荐产品

### 订单管理类
- `/api/orders` - 创建订单/获取订单列表
- `/api/orders/:id` - 获取订单详情
- `/api/orders/:id/status` - 更新订单状态
- `/api/orders/:id/logistics` - 获取物流信息

### 押金管理类
- `/api/deposit/pay` - 支付押金
- `/api/deposit/refund` - 申请押金退还
- `/api/deposit/status` - 查询押金状态

### 安装与服务类
- `/api/installation/appointment` - 预约安装
- `/api/installation/reschedule` - 修改预约时间
- `/api/service/feedback` - 提交服务反馈

### 数据统计类
- `/api/stats/transactions` - 获取交易数据
- `/api/stats/user` - 获取用户数据
